WEB2.0  :  일방향적으로 정보를 제공받던 사용자가 직접 정보를 생산하고 공유하다



HTTP : HyperText Transfer Protocol
	하이퍼텍스트 전송 규칙
HyperText 다른 페이지로 이동할 수 있는 링크를 뜻함


https://www.google.com/search?q=techit

https://	protocol

www.google.com	host	상대 서버, 혹은 컴퓨터의 주소를 의미한다

/search	path	애도 하나의 컴퓨터, 따라서 물리적인 패스가 존재

?q=techit	Query String	? 기호로 시작, &로 연결; 서버가 한정한 분야만 접속할 수 있다.



서버는 기본적으로 클라이언트와 어떤 행동을 했는지 기억하지 않는다. 이걸 쿠키를 보냄으로써 해결
따라서 민감한 데이터를 쿠키에 넣지 않도록 해야한다.

민감한 데이터를 서버가 저장하기 위해 세션이 등장, 쿠키를 보낼떄 세션 ID만 보내고 데이터는 세션에 저장


네트워크는 컴퓨터들끼리 연결된 망


공인 IP	잔체 인터넷 망에서 고유하게 식별 가능한 주소

사설 IP	공유 IP를 받은 라우터가 할당하는 IP, 모든 기기가 공유 IP를 받지 않아도 됨
		가정의 LNA과 같은 규모가 작은 네트워크에서 할당하는 주소


어떤 서비스가 최종 목적지인지 알려주는 포트, 다양한 서비스를 포트번호로 구분


dns : url을 ip 주소로 바꿔줌
